# Router rundown — seed and activation

The website's daily Intelligent Router Rundown is generated from real
benchmark research, not hand-edited fixtures. This directory contains:

- `models.json` — operator-maintained model catalog. Each row carries the
  canonical model ID, display name, provider attribution, tier, context
  window, cost signal, and aliases. Aliases let the canonicalizer fold
  AA slugs (`gpt-5-5`), HF org-paths (`moonshotai/Kimi-K2.6`), and
  OpenRouter slugs (`anthropic/claude-opus-4.7`) onto the same row.

The actual snapshot data is *not* checked in. It's generated by
`node scripts/run-research.mjs`, which:

1. Runs the production model-landscape adapters from
   `functions/lib/modelLandscape.js` (same code path the daily Cloud
   Function uses).
2. Hits Artificial Analysis, Terminal-Bench via Hugging Face, and
   Design Arena (when keys / fixtures are configured).
3. Folds the result through `lib/rundown-generator.mjs` and writes
   `src/data/router-rundown-history/<date>.json` + `latest.json`.

## Stable favorite policy

The daily rundown is a model-board artifact, not a raw benchmark table. A
board of language models runs research and analysis tasks over the source
feed, then deterministic policy turns that work into the public order:

1. GPT-5.5 xhigh
2. Claude Opus 4.7
3. GLM 5.1

Those favorites only receive policy protection while routable, flagship-tier,
freshly benchmarked, and backed by a benchmark score. A challenger must clear
both the evidence margin (`0.08`) and benchmark margin (`0.05`) on consecutive
rundowns before it can dethrone a protected favorite. The raw evidence score
stays in `score`; the displayed policy order uses `selectionScore`.

## Activation

### Artificial Analysis (live)

```bash
# Pull the key from Firebase Secret Manager (it's already bound there):
export ARTIFICIAL_ANALYSIS_API_KEY="$(firebase functions:secrets:access ARTIFICIAL_ANALYSIS_API_KEY | head -1)"

npm run research
```

Without the key, AA renders as `unavailable` — the rundown is honest
about that and suppresses tasks that have no live evidence.

### Terminal-Bench (live, no key)

Works automatically. The Hugging Face dataset endpoint is public:
`https://huggingface.co/api/datasets/harborframework/terminal-bench-2.0/leaderboard`.

### Design Arena (gated — apply for a key)

1. Apply at <https://designarena.ai/developers/apply>. Once approved
   you'll get a key starting with `sk_live_…`.
2. Add to Firebase Secret Manager so the production daily job
   automatically picks it up:

   ```bash
   firebase functions:secrets:set DESIGN_ARENA_API_KEY
   # paste the key when prompted
   firebase deploy --only functions
   ```

3. Local dev: `export DESIGN_ARENA_API_KEY=sk_live_…` before running
   `npm run research`.

While waiting on the key, an interim fixture path works for testing the
pipeline end-to-end:

```bash
export DESIGN_ARENA_FIXTURE_JSON="$(cat path/to/da-fixture.json)"
npm run research
```

Fixture shape (see `functions/scripts/test-model-landscape.mjs`):

```json
{ "data": [
  { "modelID": "anthropic/claude-opus-4.7",
    "provider": "Anthropic",
    "taskCategory": "design",
    "elo": 1480, "rank": 1, "confidence": 0.92 }
] }
```

## Production daily flow

Once secrets are bound:

1. `functions/src/scheduled.ts → refreshModelLandscapeBenchmarks` runs
   every 24 hours.
2. It calls `collectModelLandscapeBenchmarks` (writes
   `model_benchmark_snapshots/*` and `model_benchmark_source_status/*`).
3. Then `buildAndPersistRouterRundown(db, now)` re-scores and writes
   `router_rundowns/<date>` + `router_rundowns/latest`.
4. The public HTTPS function `latestRouterRundown` serves the latest
   rundown to the browser via the Firebase Hosting rewrite at
   `/api/router-rundown/<date|latest>`.
5. The website's `RouterRundownReport` component fetches that endpoint
   on hydration and replaces the build-time fallback.

If the live feed is unreachable, the page renders the build-time
fallback (whatever `npm run research` produced at last build) and
surfaces a clearly-labeled `LIVE FEED UNAVAILABLE` badge.

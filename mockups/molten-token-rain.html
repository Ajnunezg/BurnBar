Three commitments

A developer tool, not a SaaS dashboard.

OpenBurnBar is built for the kind of developer who has three agents open in parallel and only checks the bill at the end of the month.

01 / SECURITY

Local-first by default.

The canonical store is a local SQLite database, run by a local daemon under your user account. Firebase is an optional replication plane.

02 / HONESTY

Honest about confidence.

Every provider row carries one of three labels — exact, estimated, or unavailable. We don't pretend to know what the vendor doesn't tell us.

03 / REALITY

Built for multi-agent reality.

A single daemon owns routing, quota, retrieval, projects, and missions. Every surface speaks to the same local daemon.

Same model, every time

Same model.
Different account.

BurnBar's local gateway preserves the exact model you asked for. Your gpt-5.5 stays invariant. The router picks from accounts in your library that already carry that model — and if none survive, it returns a structured 503 instead of a silent substitute.
